개발자 커뮤니티 및 구인구직 웹 사이트

Project information

  • Category: Web Project
  • Language: JAVA (JDK 1.8)
  • Editor: Eclipse (Mars.2.Release(4.5.2))
  • Developer: 5명
  • Project date: 2021.07.26 - 2021.08.06

Project description

  • 프로젝트명
    개발자 커뮤니티 및 구인구직 웹 사이트
  • 목적
    기존 커뮤니티 사이트에 대상을 개발자로 특정해서 개발자 전용 기능이 구현가능한 커뮤니티를 만들었다.
  • 차이점
    사용자는 스터디 참여 및 개설, 공간대여 서비스 예약, 채용공고 지원, 사이트 이용자들의 별점 조회 등의 기능을 추가해 차별성을 높였다.
  • 담당업무
    스터디 목록, 스터디 개설 및 참가, 스터디 상세보기(차트, 지도 api), 스터디 지원자 프로필 조회, 스터디 참가자 별점으로 평가하기
기술분야 상세내용
컬렉션 ArrayList<T>, HashMap<T>
>ArrayList T 을 활용하여 더미데이터를 쉽게 활용할 수 있도록 했고, ArrayList.add / ArrayList.remove 로 데이터를 새로 추가하거나 삭제할 수 있도록 하였다.
JSTL JSTL JSP 페이지 내에서 자바 코드를 바로 사용하지 않고 로직을 내장 하였다.
JDBC JDBC 자바 파일과 데이터베이스를 연결하기 위하여 DAO 파일 내에서 사용
MVC 패턴 JSP 파일 내에서 자바 코드를 사용하지 않고 모델, 뷰, 컨트롤러로 각각 페이지를 나누어 코드를 작성함으로써 가독성을 높이고, 오류 발생시 오류 수정에 시간을 아낄 수 있도록 함
jQuery HTML의 클라이언트 사이드 조작을 단순화 하도록 설계된 크로스 플랫폼의 자바스크립트 라이브러리
Kakao API 지도 API
느낀점
처음 사용해보는 JSP Servlet기반의 Model2웹 프로젝트 였습니다. 스터디 목록에 페이징기능과 검색기능을 추가해 사용자의 편의를 고려했는데 검색된 페이지에서 다 른 페이지로 이동했을때 페이징 오류가 났습니다. 디버깅을 통해서 sql문의 rownum의 특성을 고려하지 못해 발생한 오류임을 알 수 있었습니다.
웹 프로젝트 특성상 프론트엔드, 백엔드의 언어를 모두 사용해서 그동안 배웠던 언어와 기능들을 다양하게 활용해서 페이지를 구현했기 때문에 하나의 페이지를 만드는데도 다양한 오류에 직면하게 되어 소요되는 시간과 작업량이 많다는 것을 몸소 체험할 수 있었습니다.
DAO, DTO를 사용해서 데이터를 관리했는데 문법이나 기술등은 크게 어렵진 않았지만 비슷한 코드의 반복이 계속되는 점이 아쉬웠습니다. 이번 프로젝트에서는 팀원들과의 코드를 통합하기 위해서 GIT으로 소스파일을 관리해서 이전 프로젝트보다 코드공유를 수월하게 진행할 수 있었고, 단축된 시간만큼 각자 구현에 힘 쓸 수 있었습니다.
